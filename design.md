# 沉浸式 3D 摇骰子工具 - 设计方案

> **项目类型：** 微信小程序（单机工具类）→ H5 版本先行
> **目标场景：** 朋友聚会（主要是大话骰/吹牛玩家）
> **核心功能：** 模拟 3D 摇骰子，作为酒桌上的隐私工具（不联机）
> **视觉风格：** 3D 拟真、沉浸式。模拟一个深绿色毛毡赌桌，配象牙白质感骰子。

---

## 一、界面布局 (UI Structure)

设计在一个竖屏页面内完成：

### 1. 顶部状态栏 (HUD)
- 在顶部有一个半透明的浮层，显示「摇出时间：HH:mm:ss」
- **字体需求：** 绿色 LCD/电子表风格数码管字体

### 2. 核心显示区 (3D Canvas)
- 占据屏幕大部分，使用深绿色毛毡纹理作为背景
- 显示 N 个（默认5个）3D 骰子
- **骰子摆放：** 严禁整齐排列。每次摇完后，骰子的坐标和旋转角度必须是随机生成的，呈现堆叠、倾斜、散乱的「杂乱感」
- **骰子材质：** 象牙白，圆角。1点为大红色，2-6点为深黑色

### 3. 底部操作区
- **「摇（ROLL）」按钮：** 位于底部正中心。一个巨大的、圆形的、拉丝金属质感按钮，带有下沉的物理点击阴影
- **设置图标：** 位于底部右下角。一个细线条、半透明的「齿轮」图标

## 二、功能需求与逻辑实现

| 功能模块 | 详细逻辑 | 备注 |
| :--- | :--- | :--- |
| **3D 加载** | 在页面加载时，初始化 WebGL 容器，加载 3D 骰子模型和毛毡纹理 | H5 使用 Three.js |
| **「摇」操作** | 点击按钮触发。禁用按钮，防止连点 | - |
| **3D 动画** | 骰子在屏幕内剧烈翻滚（模拟碰撞和重力，无需复杂物理引擎，可用随机冲力和预设乱序坐标） | - |
| **音效/震动** | 动画开始时循环播放「哗啦哗啦」碰撞声音。同时循环调用短震动API | H5 使用 Vibration API 和 Web Audio |
| **停止/结果** | 动画停止（例如 1.5 秒后）。骰子停在最终随机点数。更新顶部的「摇出时间」 | - |
| **设置面板** | 点击齿轮图标，当前页面弹出一个毛玻璃效果（Glassmorphism）的浮层（Modal），不跳转新页面 | - |
| **自定义数量** | 设置面板内提供滑块或步进器（范围 1-10），调节主界面骰子数量 | - |
| **常亮设置** | 设置面板内提供开关 | H5 使用 Screen Wake Lock API |

## 三、核心素材资产清单 (Assets)

1. **3D 模型：** 标准六面骰子模型（程序生成，无需外部文件）
2. **材质贴图：** 
   - 骰子材质：象牙白，圆角，1点红、2-6点黑
   - 背景：深绿色毛毡纹理
3. **音效：** 骰子在骰盅内碰撞的循环音频文件

---

## 四、技术栈

### H5 版本
- **3D 渲染：** Three.js
- **物理效果：** 简化动画（随机冲力 + 预设坐标），无需 Cannon.js 等物理引擎
- **UI：** 原生 HTML/CSS，毛玻璃效果使用 `backdrop-filter`
- **音效：** Web Audio API
- **震动：** Navigator.vibrate()
- **屏幕常亮：** Screen Wake Lock API

### 小程序版本（后续）
- **3D 渲染：** 微信 XR-Frame 或适配版 Three.js
- **API：** wx.vibrateShort, wx.setKeepScreenOn, wx.createInnerAudioContext

## 六、动画规格

### 摇骰子动画
- **时长：** 1.5 秒
- **运动方式：** 
  - 骰子从中心点向四周散开
  - 伴随随机旋转（每个轴 720° ~ 1440°）
  - 模拟重力下落效果
- **缓动：** ease-out（前半段快速，后半段减速）

### 停止状态
- 骰子随机散落在屏幕中央区域
- 每个骰子有轻微的随机倾斜角度（-15° ~ 15°）
- 允许轻微重叠，呈现自然感
