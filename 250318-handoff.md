## 1. 当前任务目标

开发一个沉浸式 3D 摇骰子 H5 应用，用于朋友聚会大话骰/吹牛场景。

**核心需求**:
- 竖屏 H5 页面（后续计划转为微信小程序）
- 3D 骰子渲染（Three.js）
- 1-10 个骰子可调
- 真实音效 + 音画同步
- 简洁 UI，沉浸式体验

**完成标准**:
- 点击"摇"按钮触发 1.5 秒动画
- 骰子随机散落且不重叠
- 动画结束音效立即停止
- 设置面板支持调节骰子数量和开关

## 3. 关键上下文

### 技术栈
- **3D 引擎**: Three.js (CDN v0.160.0)
- **物理**: 无物理引擎，使用简化动画（随机冲力 + 预设坐标）
- **音效**: HTML5 Audio API (非 Web Audio)
- **UI**: 原生 CSS，毛玻璃效果用 `backdrop-filter`

### 用户明确要求
- 骰子数量: 1-10 个（默认 5 个）
- 音效来源: 从提供的视频 `骰子声音.mp4` 提取
- 不需要陀螺仪/摇一摇功能
- 不要"摇出时间"显示（已移除）

### 关键设计决策
- 骰子使用程序生成（BoxGeometry + Canvas 贴图），非 GLB 模型
- 背景图使用 `Gemini_Generated_Image_ua9fvkua9fvkua9f.png`（深绿色毛毡）
- 音效循环播放，动画结束强制停止（音量归零）

### 已知约束
- H5 版本先行，后续计划转微信小程序
- 竖屏 only
- 需要用户交互后才能播放音频（浏览器策略）

## 4. 关键发现

### 核心算法
**防重叠位置生成** (`js/dice.js:211-258`):
```javascript
minDistance = 1.6  // 骰子大小的 1.3 倍以上
maxAttempts = 100  // 防止无限循环
```

**动画缓动** (`js/main.js:320`):
```javascript
easeProgress = 1 - Math.pow(1 - progress, 3)  // ease-out
```

### 音画同步关键点
- `audio.js:75-85` 使用强制静音 + 延迟恢复音量确保立即停止
- `main.js:309` 动画结束调用 `stopRollingSound()`，不播放落地音效

### Three.js 兼容性处理
- `SRGBColorSpace` 不存在时回退到 `sRGBEncoding`
- 背景图加载失败时使用纯色 fallback (#1a472a)

## 7. 风险与注意事项

### 已验证（不建议重复尝试）
- ✅ 物理引擎（Cannon.js）非必要，简化动画足够
- ✅ 音效用 HTML5 Audio 比 Web Audio 更简单可靠
- ✅ 程序生成骰子比 GLB 模型加载更快

### 易踩坑点
- ⚠️ 音频播放必须在用户交互后（已处理，但需注意）
- ⚠️ Three.js 版本差异（`SRGBColorSpace` 不存在于旧版本）
- ⚠️ 骰子数量太多（10个）时位置生成可能失败（已设置尝试次数上限）

- ❌ 添加陀螺仪（用户明确不要）
